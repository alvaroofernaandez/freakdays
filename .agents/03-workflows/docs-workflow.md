# Workflow: Documentación

1. Identificar secciones impactadas (`.agents/`, `docs/`, `README`).
2. Validar consistencia con `AGENTS.md` y arquitectura actual.
3. Actualizar referencias cruzadas y rutas.
4. Verificar que el contenido refleje el estado real del frontend y su relación con `freak-days-api`.
5. Abrir PR de docs con etiqueta de riesgo adecuada (`risk/low` por defecto).

## Criterio de calidad

No dejar placeholders vacíos ni secciones ambiguas.
