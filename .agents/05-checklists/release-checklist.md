# Release Checklist

- [ ] Scope de release confirmado (issues incluidos y excluidos).
- [ ] Riesgos altos/críticos revisados por responsables.
- [ ] Validación de regresión ejecutada para flujos principales.
- [ ] Quality gates de PR en verde para cambios incluidos.
- [ ] Cambios de auth/tenant revisados con foco de seguridad.
- [ ] Contratos FE↔BE sincronizados con `freak-days-api`.
- [ ] Plan de rollback documentado y factible.
- [ ] Estado de documentación actualizado (`README`, `docs`, `.agents` si aplica).
- [ ] Comunicación de release preparada (impacto, riesgos residuales, acciones post-release).
